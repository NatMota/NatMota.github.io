Trying out compact LLMs for product work

Trying out compact LLMs for product work

Motivation

I wanted to recreate a personal assistant environment for product managers who:

  • Do not have access to, or are not allowed to place sensitive data on online services

  • Want to use software for free

  • Know python but have no time to learn related tools

  • May be concerned with software licensing restrictions

  • Are curious about LLM's and are exposed to them already

  • Do not have a powerful machine to run LLM's

  • Want to automate the crappy first version of a doc and overcome writer's block faster

Mise en place

I'm using an apple silicon mac. Use ChatGPT or Claude to guide you through system specific setup.

  • Computer with about 1.5GB of disk space, about 2GB of free ram, CUDA gpu a plus

    • Used a Mac air M1 fine.

  • Python IDE of your choice

  • Create a new python environment:

    • python -m venv llm_env

    • source llm_env/bin/activate

    • python -m pip install --upgrade pip

    • pip install transformers torch

  • Essential facts and lexicon

    • Large language models (LLM's) are systems specialised in handling language by

      • 1. Being painstakingly baked to generate 'correct' language patterns

      • 2. Taking a language input and converting it into mathematical symbols

      • 3. Layer the next best mathematical symbols based on baking presets and convert the result into language output

    • ChatGPT is an LLM that has an additional layer of frosting to specifically work better in two way conversations

    • Language input into an LLM is a prompt

    • The prompt becomes the seed not just for the answer, but for the answering itself. It can have a dramatic effect on the quality of the response, and has led to the rise of the field of 'prompt engineering'

    • ' Zero-shot prompting directly instructs the model to perform a task without any additional examples to steer it

    • LLM's are baked to answer a question 'correctly' by matching the core concepts of the prompt to trained language structures

      • This means that much like a conversation, the output will never be exactly the same across two equal prompts, except for example when working on coding prompts due to much more predictable language structures in coding

    • Prompts are converted into tokens. As a simple example, every word in a prompt will be matched to a token. Imagine tokens as numbers that correspond to a specific word.

    • LLM's can be made more effective by:

      • improving the amount and quality of language data available to condition them

      • improving their ability to detect language structures and hierarchy, their depth

      • improving their ability to process more information in each conversation or prompt, their width

    • One simple way to compare how much information LLM's can meaningfully process in a conversation or a prompt response is to compare their context window

      • A context window is the limit in the chain of tokens that an LLM can hold at once when replying in a conversation or prompt, including the conversation history AND response

      • For example, Chat GPT's free version has a context window of 8k tokens, but the enterprise version is 128k tokens - This means it can operate between:

        • 12-13 pages of single-spaced text in the free version if you imagined it as one doc

        • 197 pages doc on the enterprise version

    • Commercial conversational LLM's are extensively refined and factually enriched i.e. with true historical events and mathematical reasoning... which means a lot of manual work goes into it and gives you an inflated impression of their raw ability!

      • For this alone it is very educational to play with non-tuned LLM's

    • An introduction I like from S Wolfram on the subject of LLMs is available here

Seasonal sauce making

This technology is moving very fast, so make sure your pick fresh LLM's for your work.

I have sourced LLM's from Hugging Face. This is the go-to place to learn about new models and compare them.

We want to be able to run this in any machine where you're allowed to run python no matter the corporate environment. This approach just needs python, and the models I used here are open. I have only used them for personal reasons and you may still need to verify the license requirements for your specific use case. To the best of my ability, no data is then sent online with this approach. So hopefully it's a quick and safe solution to get you going.

I used Hugging Face's open LLM scoreboard here.

I picked HF's own Smol 360M model. The number means it has 360 Million adjustments to the neurons of the model, assembled in the structure described here. I picked it because:

  • This model is free to use commercially

  • This model is fairly small

  • This model is quite fresh, less than a month old at writing

  • Other factors that make this 'open' are the tools needed to reconstruct it

    • The training data is available online

    • The model weights are available to be downloaded - this is crucial to being able to modify the model from different save points at which it was created

      • Closed models like ChatGPT do not make their weights available

    • Other supporting information needed to create output is available

  • Online demo available here

  • It has a context window of 2k tokens, which is about 5-6 pages of information. Not bad!

Hugging Face is great to help you explore models. You can also get licensing for other more closed and more advanced models, and it's the go-to place for the community to release new models and compare their effectiveness using standard tests.

This made me realise the heft of LLM's. A rough rule of thumb is that you need 2X GB of disk and RAM to run a model with X billion parameters. Even more to bake a new one. So I might be able to run Llama 7b on my gaming rig, but no chance with the 405b version... So to get a GPT4 competitor in a machine you'd need 900GB of space, 1 TB of RAM, and strong video card capabilities.

Note how there are two versions of Smol 360M:

  • ' 360M LLM

  • ' 360M LLM - instruct version

    • This is 360M model with a conversational layer

    • In the same way, the ChatGPT model can be ported to GPT2, GPT3, and could even be ported to a nonGPT LLM, possibly

    • I implemented it with a chat runtime using code suggested by Claude and it had pretty disappointing results, so I stopped using it. Closed source conversation layers are a big market advantage at the moment!

For example if you wanted to try something like Google Gemma, you need to accept their terms of use in HF and then authenticate at runtime to be able to load and run their model.

Roasting (my CPU)

Get a code sample from ChatGPT or Claude. This worked on my machine:

# pip install transformers

# pip install torch==2.2.0 - compatible with M1

from transformers import AutoModelForCausalLM, AutoTokenizer

checkpoint = "HuggingFaceTB/SmolLM-360M"

device = "cpu" # for GPU usage or "cpu" for CPU usage

tokenizer = AutoTokenizer.from_pretrained(checkpoint)

# for multiple GPUs install accelerate and do `model = AutoModelForCausalLM.from_pretrained(checkpoint, device_map="auto")`

model = AutoModelForCausalLM.from_pretrained(checkpoint).to(device)

test_intro_prompt_3 = f"""Write an article about the benefits of meditation.

Introduction:[Write an introduction about meditation.]

Benefits:

1. [Benefit 1]

2. [Benefit 2]

3. [Benefit 3]

Conclusion:[Write a conclusion about the benefits of meditation.]

inputs = tokenizer.encode(test_intro_prompt_3, return_tensors="pt", truncation=True, max_length=2000).to(device) outputs = model.generate( inputs, max_new_tokens=300, temperature=0.2, top_p=0.95, do_sample=True )print(tokenizer.decode(outputs[0]))

Now save this as a .py file and run it on your terminal:

(llm_env) (base) ... personal LLM % python smol.py
